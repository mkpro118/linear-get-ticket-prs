# Release Notes

```
Generate human-readable release notes between two git refs

Usage: release-notes --base <BASE> --head <HEAD> --config-key <CONFIG_KEY> --repo-name <REPO_NAME>

Options:
      --base <BASE>              The base git ref (must be an ancestor of head)
      --head <HEAD>              The head git ref
      --config-key <CONFIG_KEY>  Git config key prefix that maps branch namespaces to GitHub handles
      --repo-name <REPO_NAME>    Repository/org name to match in merge commit subjects
  -h, --help                     Print help

```
